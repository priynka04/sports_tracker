# рЯОѓ Multi-Object Detection and Persistent ID Tracking

A computer vision pipeline that detects every person in a video and assigns each one a **unique persistent ID** that stays consistent across the entire video вАФ even during movement, occlusion, and crowded scenes.

Built using **YOLOv8** for detection and **DeepSORT** for tracking.

## рЯ§Ц Model and Tracker Details

### Detector вАФ YOLOv8 Nano
- **What it does:** Scans each video frame and detects all people
- **Why chosen:** Fast, accurate, works well on CPU, single-line setup
- **Model file:** `yolov8n.pt` вАФ downloads automatically on first run (~6 MB)
- **Detection class:** Person only (class 0 from COCO dataset)
- **Confidence threshold:** 0.40 (detections below 40% are ignored)

### Tracker вАФ DeepSORT
- **What it does:** Assigns unique IDs and keeps them stable across frames
- **Why chosen:** Industry-standard baseline, handles occlusion well, no training needed
- **How it works:**
  - **Kalman Filter** вАФ predicts where each person will be in the next frame
  - **Hungarian Algorithm** вАФ matches new detections to existing tracks optimally
  - **Cosine Distance** вАФ uses visual appearance to re-identify people after occlusion
- **Max age:** 30 frames вАФ a track is deleted if no match found for 30 consecutive frames

## рЯУК Optional Enhancements

### 1. Trajectory Visualization
Each tracked person has a colored trail line showing their movement path drawn directly on the output video.

### 2. Movement Heatmap
A color-coded overlay showing which areas of the scene had the most human activity. Red = high activity, Blue = low activity.

### 3. Object Count Over Time
A time-series graph showing how many people were actively tracked at each second of the video, with average line marked.

### 4. Evaluation Metrics
Automatically computed statistics including:
- Total unique IDs detected
- Average and maximum active tracks per frame
- Detection rate (percentage of frames with detections)
- Average track length (measure of ID stability)

## вЪ†пЄП Known Limitations

| Limitation | Explanation |
|-----------|-------------|
| ID switching | When two people cross paths or heavily overlap, DeepSORT may briefly swap their IDs |
| Re-entry IDs | A person who leaves and re-enters the frame gets a new ID each time |
| Distant subjects | Small/distant people may not be detected by the Nano model |
| CPU only | No GPU used вАФ processing is slower than real-time |
| Similar appearance | People with very similar clothing may confuse the appearance matcher |

---

## рЯЪА Possible Improvements

- Use **ByteTrack** or **BoT-SORT** instead of DeepSORT for more stable ID assignment
- Use **YOLOv8m** (medium) instead of Nano for better detection accuracy
- Add **GPU support** for real-time processing speed
- Add **Re-ID model** (e.g. OSNet) for better appearance matching across long gaps
- Add **team color clustering** to group players by team in sports footage
